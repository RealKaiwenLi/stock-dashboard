# 每日持仓起算日

## 背景

当前 Nasdaq guide 每日持仓从下载到的全部历史日线起始处连续模拟，因此会继承 2026-07-01 之前形成的持仓。用户希望生产每日持仓从 2026-07-01 重新起算。

## 目标

- 支持在模型版本中配置持仓起算日。
- 指标仍使用起算日前的历史数据计算，避免 MACD 和 EMA 从零预热。
- 仅重置持仓状态，不继承起算日前的持仓或待执行订单。
- 起算日初始持仓为模型的 `signal_symbol`。
- 起算日收盘开始判断信号，继续遵循 T 日确认、T+1 开盘成交。
- 每日报告明确展示持仓起算日和实际使用的首个交易日。

## 非目标

- 不修改 MACD、退出 EMA 或 Hist 规则。
- 不回写或删除既有 Notion 每日报告。
- 不让 GitHub Action 自行维护跨日状态；每次仍从历史行情确定性重算。

## 验收标准

1. 模型未配置起算日时，结果与现有连续历史口径一致。
2. 配置 `2026-07-01` 时，起算日前的交易信号不影响起算日初始持仓。
3. 指标计算仍包含起算日前的数据。
4. 起算日不是交易日时，从之后第一个可用交易日开始。
5. 起算日晚于最新行情时返回明确错误。
6. 当前生产模型升级版本并配置 `2026-07-01`。
7. Markdown 和 Notion 信号表显示持仓起算口径。
