# Agent Instructions

## Project

This is an AI-native development package built as a Vite React project.

Package manager:

- `npm`

This project allows agents to read and write files inside:

- `/Users/kaiwenli/Desktop/stock-dashboard`

## Context Router

Before planning or implementation, agents should read:

- `docs/project-context.md` for stable project context.
- `AGENTS.md` for workflow, permissions, and role boundaries.

For feature work, agents should also read the relevant `brd/` documents:

- `./brd/<feature>.md`
- `./brd/<feature>.tech.md`
- `./brd/<feature>.tasks.md`

Keep `AGENTS.md` compact. Put stable project context in `docs/project-context.md` and feature-specific context in `brd/`.

## Workflow

The expected development flow uses specialized agents:

1. The user asks the PM agent to write or update the BRD.
2. The PM agent writes the BRD under `./brd`.
3. The user asks the Technical Planner agent to convert the BRD into implementation artifacts.
4. The Technical Planner agent writes `./brd/<feature>.tech.md` and `./brd/<feature>.tasks.md`.
5. The user asks the TDD agent to execute the code change.
6. The TDD agent reads the BRD, technical plan, and task list, then implements tasks in order with tests and verification.

Expected feature document set:

- `./brd/<feature>.md`: product requirements from the PM agent.
- `./brd/<feature>.tech.md`: technical plan from the Technical Planner agent.
- `./brd/<feature>.tasks.md`: dependency-ordered task list from the Technical Planner agent.

## Agent Roles

PM agent:

- Owns product requirements and behavior definitions.
- Writes BRD documents under `./brd`.
- Should make acceptance criteria concrete enough for implementation and testing.
- Should not make code changes unless the user explicitly asks.

Technical Planner agent:

- Owns technical planning and implementation task breakdown.
- Reads the relevant BRD in `./brd` and inspects the current codebase before making technical decisions.
- Writes `./brd/<feature>.tech.md` with the technical approach, library choices, file plan, testing strategy, risks, and open questions.
- Writes `./brd/<feature>.tasks.md` with dependency-ordered tasks that the TDD agent can execute.
- Should not write production code or install dependencies.
- Should mark unclear product or technical decisions as `NEEDS CLARIFICATION`.
- Should recommend new libraries only when they clearly reduce complexity, and should flag them for user approval.

TDD agent:

- Owns implementation, tests, and verification.
- Uses the latest relevant `./brd/<feature>.tasks.md` as the execution plan.
- Uses `./brd/<feature>.md` and `./brd/<feature>.tech.md` as supporting context.
- Should ask for clarification before coding if the task list, technical plan, or BRD is ambiguous or contradictory.
- Should prefer a TDD flow when practical.
- Should execute tasks in order unless a task is explicitly marked parallelizable with `[P]`.
- Should run the configured verification commands before finishing.

## Commands

Development server:

- `npm run dev`
- This is long-running, so ask before starting it unless the user explicitly requested a dev server.

Build command:

- `npm run build`

Lint command:

- `npm run lint`

Test command:

- `npm test`
- Runs Vitest once with `vitest run`.

## Internationalization

- Add or update all user-facing Chinese and English UI text in `src/i18n/translations.js`.
- Do not hardcode translated UI strings in React components or service modules.
- Keep the `zh` and `en` translation structures aligned and update `src/i18n/translations.test.js` when adding translation keys or localization behavior.

Test watch command:

- `npm run test:watch`
- This is long-running, so ask before starting it unless the user explicitly requested watch mode.

Agents may run these commands without asking first:

- `npm run build`
- `npm run lint`
- `npm test`

After running Python syntax checks, agents should delete any `__pycache__/` directories or `.pyc` files generated by that check without asking first.

Ask the user before:

- Installing or updating dependencies, including `npm install ...`
- Deleting files or directories
- Running destructive git commands, such as `git reset` or `git checkout --`
- Reading from or writing to files outside this project directory
- Starting long-running servers, such as `npm run dev`, unless the user requested it
- Starting long-running watch commands, such as `npm run test:watch`, unless the user requested it

When working on features, prefer a TDD flow when practical:

1. PM agent writes or updates `./brd/<feature>.md`.
2. Technical Planner agent writes or updates `./brd/<feature>.tech.md` and `./brd/<feature>.tasks.md`.
3. TDD agent reads the task list and adds or updates a focused test for the current task.
4. TDD agent runs the relevant test command and confirms the expected failure when practical.
5. TDD agent implements the smallest change that satisfies the test.
6. TDD agent runs `npm test`, `npm run build`, and `npm run lint` when relevant.
